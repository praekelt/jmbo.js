Jmbo apps

Introduction

Jmbo apps doesn't want to be a framework. However I don't know how one goes about providing a collection of "things" that formalizes a method of building HTML5 apps without becoming a framework.

So whilst one of my goals is not to be a framework I think it's inevitable. That said: We should try and be clear and simple in our implementations and minimal in css declarations.

But not too simple and not too minimal. It's meant to increase productivity afterall. :P

Requirements

I hope we don't ever require more than this

  1. Underscore
  2. Backbone
  3. Zepto or jQuery

Getting started...

Apps don't generally refresh when a new page is load, rather new pages (or views) are loaded when required and transitioned into the viewport.

Jmbo provides two mechanisms in which pages can transition (not loadable yet), the first provides a drill down hierarchy: `jmbo.navigation.ControllerView` and the second allows you to switch the views context `jbmo.navigation.TabBarControllerView.` 

    var navController = new jmbo.navigation.ControllerView;
    $('#container').html(navController.render());

    // define your own view.
    var menuView = Backbone.view.extend({
        render: function() {
            $el.html('pew pew pew');
            return this.el;
        }
    });

    // place you view in a viewController
    var menuViewController = jmbo.view.Controller({
        childView: menuView,
        title: 'Jmbo Apps!'
        /*
        animation: 'slide',
        titleView: new jmbo.view.TitleView,
        */
    });

    // push your view onto the navigationControllerView.
    navController.pushViewController(menuViewController);

});
